# 关于打卡(tick)板块的说明
打卡（tick）是一个跟踪习惯养成的板块。你可以创建一个打卡任务，我们会设置它如下属性：
* 标题，例如跳绳
* 描述，任务的详细描述，支持markdown
* 周期频率，可以是每天/每周/每月
* 周期里的具体执行时间，例如每天晚上8点、每周一（可以再包含具体时间），每月第8天（可以再包含具体时间）
* 开始时间和结束时间，这个打卡任务从什么时候开始，持续到什么时候结束
* 打卡质量，可选项，在打卡后，用户可以评价这次打卡质量如何，例如打卡完健身，可以自己评价一下完成质量，从一星到五星
* 每次打卡获得积分，这是可选项，并且它支持设置阶梯得分，例如完成第一天打卡得1分、连续完成第2天得2分、连续完成第3天得3分、第3天后每次打卡都得3分，但若中断一天，则重新计算，打卡第一天重新再得1分、第2天得2分...

有了打卡任务后，用户需要按周期进行打卡，每次打卡产生一条打卡日志，日志有以下属性：
* 打卡时间，什么时间完成了这个打卡任务
* 打卡质量，如果有配置这个选项，那打完卡后可以自己评价，从一星到五星
* 备注，可选项，可以给这次打卡加个备注，例如这次打卡收获很大，支持markdown格式
* 打卡获得积分，此次打卡得到多少分，同时展示本任务累计已经获得多少积分

一些注意项：
* 如果没到时间点不能打卡，例如一个每日打卡的任务，打完今天的就显示今日已打卡，到了明天（0:00），才会变成又可以打卡的状态
* 允许撤销打卡，撤销打卡后相当于把这条打卡日志删除掉，如果有增加的分数也要重置回去；撤销打卡后可以重新打卡
